# Phase 3 — Improvement Synthesis

**Prerequisite:** Phase 1 audit + Phase 2 research.

**Goal:** Merge findings and research into a prioritized, per-module improvement plan the owner can execute. Honour completeness-with-focus: **demote before delete**.

## Method

For every Phase 1 finding, decide one disposition and justify it against the hero spine:

- **KEEP** — leave as is (usually a WORKS note that research confirms is competitive).
- **SIMPLIFY** — reduce, reorder, progressively disclose; cite the borrowed pattern from Phase 2.
- **REBUILD** — UX or backend rework; state the target behaviour concretely and testably.
- **DEMOTE** — move behind the spine (secondary nav, detail view, "advanced"); this is the default for SECONDARY-TO-SPINE.
- **CUT** — last resort; must carry an explicit spine justification.

## Output

- **The spine, sharpened** — the hero journey after improvements, in one arrow chain.
- **Per-module plan** — table: `Finding → Disposition → Concrete change (testable) → Borrowed pattern (if any) → Effort (S/M/L) → Priority`.
- **Sequenced roadmap** — what to do first for maximum demo/pitch payoff per unit effort. Three tiers: *pitch-critical now / strong-signal next / post-pitch polish*.
- **Explicitly list anything demoted or cut**, so nothing is silently removed.

Where a change touches money, permissions, or irreversible state, state the change in testable terms and flag any product decision that still needs the owner. Do not resolve those by picking the least-work option.
